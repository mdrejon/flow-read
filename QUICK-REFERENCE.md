# Quick Reference: Composer Autoload & Namespacing

## ✅ What Was Set Up

1. **composer.json** - Defines PSR-4 autoloading: `FlowRead\` → `app/`
2. **vendor/autoload.php** - Auto-generated by Composer
3. **Namespaced Classes** - Organized in `app/` directory
4. **Main Plugin File** - Updated to use autoloader

## 🚀 How to Use

### Creating a New Class

1. **Choose location based on namespace**:
   - Admin features → `app/Admin/`
   - Frontend features → `app/Frontend/`
   - Utilities → `app/Helpers/`
   - Business logic → `app/Services/`

2. **Create file with proper namespace**:

```php
<?php
// File: app/Services/ReaderService.php

namespace FlowRead\Services;

class ReaderService {
    public function calculate_reading_time( $content ) {
        // Your code here
    }
}
```

3. **Use the class anywhere**:

```php
use FlowRead\Services\ReaderService;

$reader = new ReaderService();
$time = $reader->calculate_reading_time( $content );
```

## 📁 Current Structure

```
FlowRead\
├── Plugin                    (Main plugin class)
├── Admin\
│   └── Admin                 (Admin functionality)
└── Frontend\
    └── Frontend              (Frontend functionality)
```

## 🔄 After Adding New Classes

```bash
composer dump-autoload
```

## 💡 Key Points

- **Namespace = Directory path** from `app/`
- **Class name = File name**
- **No manual `require` needed**
- **Import with `use` statements**

## Example: Add a Settings Class

**File**: `app/Admin/Settings.php`
```php
<?php
namespace FlowRead\Admin;

class Settings {
    public function register_settings() {
        // Settings code
    }
}
```

**Use in Plugin.php**:
```php
use FlowRead\Admin\Settings;

new Settings();
```

That's it! The autoloader handles everything automatically. 🎉
