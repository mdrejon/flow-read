# FlowRead - Composer Autoload Setup

## Overview
This plugin now uses Composer's PSR-4 autoloading with proper namespacing.

## Namespace Structure
- **Root Namespace**: `FlowRead\`
- **Namespace to Directory Mapping**: `FlowRead\` → `app/`

## Installation

### 1. Install Composer Dependencies
Run this command in the plugin directory:

```bash
composer install
```

Or if you want to install without dev dependencies:

```bash
composer install --no-dev
```

### 2. Generate Autoload Files
If you add new classes, regenerate the autoload files:

```bash
composer dump-autoload
```

## How to Create New Classes

### Example 1: Create a Helper Class

**File**: `app/Helpers/StringHelper.php`

```php
<?php
namespace FlowRead\Helpers;

class StringHelper {
    public static function sanitize_title( $title ) {
        return sanitize_title( $title );
    }
}
```

**Usage**:
```php
use FlowRead\Helpers\StringHelper;

$clean_title = StringHelper::sanitize_title( 'My Title' );
```

### Example 2: Create a Service Class

**File**: `app/Services/ReadingService.php`

```php
<?php
namespace FlowRead\Services;

class ReadingService {
    public function get_reading_time( $content ) {
        $word_count = str_word_count( strip_tags( $content ) );
        $reading_time = ceil( $word_count / 200 );
        return $reading_time;
    }
}
```

**Usage**:
```php
use FlowRead\Services\ReadingService;

$service = new ReadingService();
$time = $service->get_reading_time( $post_content );
```

## Namespace Rules

1. **Namespace must match directory structure**
   - `app/Admin/Settings.php` → `namespace FlowRead\Admin;`
   - `app/Services/Cache/Manager.php` → `namespace FlowRead\Services\Cache;`

2. **Class name must match filename**
   - File: `Admin.php` → Class: `Admin`
   - File: `ReadingService.php` → Class: `ReadingService`

3. **Use proper imports**
   ```php
   use FlowRead\Admin\Admin;
   use FlowRead\Services\ReadingService;
   ```

## Benefits

✅ **PSR-4 Autoloading**: No need to manually require files
✅ **Organized Structure**: Clear separation of concerns
✅ **Namespace Collision Prevention**: Avoid conflicts with other plugins
✅ **Modern PHP**: Follow industry-standard practices
✅ **Easy Testing**: Better for unit testing
✅ **Dependency Management**: Use Composer packages if needed

## Adding Composer Packages

You can add third-party packages:

```bash
composer require vendor/package-name
```

Example:
```bash
composer require monolog/monolog
```

Then use in your code:
```php
use Monolog\Logger;
use Monolog\Handler\StreamHandler;

$log = new Logger('flowread');
$log->pushHandler(new StreamHandler(FLOWREAD_PLUGIN_DIR . 'logs/app.log', Logger::WARNING));
```

## Development vs Production

**Development** (includes dev dependencies):
```bash
composer install
```

**Production** (optimized, no dev dependencies):
```bash
composer install --no-dev --optimize-autoloader
```

## Troubleshooting

### Autoload not working?
1. Make sure vendor/autoload.php exists
2. Run `composer dump-autoload`
3. Check namespace matches directory structure
4. Verify class name matches filename

### Class not found?
1. Check namespace declaration in the file
2. Ensure proper PSR-4 structure
3. Regenerate autoload: `composer dump-autoload`
